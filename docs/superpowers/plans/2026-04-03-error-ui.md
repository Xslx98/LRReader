# Error UI 改进 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 阻止 HTTP 错误响应体（HTML）泄露到用户可见的错误消息中，并将分类列表的错误状态替换为图标 + 友好文字的统一风格视图。

**Architecture:** 两层改动。Layer 1：在 `ensureSuccess()` 移除响应体拼接，确保异常消息始终为纯友好文字。Layer 2：新建可复用 `view_error_state.xml` 布局，替换 `LRRCategoriesScene` 的纯文本 tip，分别用 `showError()` 和 `showEmpty()` 处理两种状态。

**Tech Stack:** Kotlin（`LRRApiUtils.kt`）、Java（`LRRCategoriesScene.java`）、Android XML Layout、MockWebServer（测试）

  ```kotlin
  internal fun ensureSuccess(response: Response) {
      if (!response.isSuccessful) {
          val friendlyMsg = when (response.code) {
              401, 403 -> "认证失败，请检查 API Key 是否正确"
              404 -> "资源未找到 (404)"
              in 500..503 -> "服务器错误 (${response.code})，请稍后重试"
              else -> "请求失败 (HTTP ${response.code})"
          }
          throw IOException(friendlyMsg)
      }
  }
  ```

  ```java
      if (mErrorView != null) {
          mErrorView.setVisibility(View.VISIBLE);
  ```

## 验收检查

完成后，手动验证：

1. **服务器错误**：在 Categories 场景断开网络或设置错误服务器地址，刷新应看到红色感叹号图标 + "出错了" 标题 + 友好错误描述 + "重试"按钮
2. **重试功能**：点击"重试"按钮触发重新加载
3. **空列表**：服务器正常但无分类时，显示纯文字提示（无图标、无按钮）
4. **无 HTML**：任何错误场景均不出现 `<html`、`<body>` 等标签
5. **深色主题**：切换深色主题后颜色正确（文字、背景自动适配 `?attr/` 属性）
